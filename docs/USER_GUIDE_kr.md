## Introduction (소개)

### 마케팅 믹스 모델링(MMM)이란?

마케팅 믹스 모델링은 다음을 이해하는 데 도움을 주는 통계적 기법입니다:

  - **각 마케팅 채널이 매출에 얼마나 기여하는가?**
  - **채널 간 최적의 예산 배분은 무엇인가?**
  - **채널의 포화 시점(수확 체감)은 언제인가?**
  - **광고의 이월 효과(Adstock)는 어떠한가?**

### 이 도구를 사용해야 할 때

✅ **다음과 같은 경우 Ridge MMM을 사용하세요:**

  - 1\~3년 치의 주간(Weekly) 마케팅 데이터가 있을 때
  - 2\~10개의 마케팅 채널에 비용을 집행할 때
  - 빠르고 실행 가능한 예산 추천이 필요할 때
  - "만약에(What-if)" 시나리오를 비교하고 싶을 때

❌ **다음과 같은 경우 Ridge MMM을 사용하지 마세요:**

  - 데이터가 6개월 미만일 때 (신뢰할 수 있는 추정을 하기에 부족함)
  - 실시간 최적화가 필요할 때 (이 도구는 전략적 계획용입니다)
  - 채널이 20개 이상일 때 (이 경우 계층적 베이지안 MMM을 고려하세요)

## Quick Start (빠른 시작)

### 1단계: 데이터 준비

다음 구조로 CSV 파일을 생성하세요:

```csv
date,revenue,google_uac,meta,apple_search,tiktok
2023-01-01,15000000,2000000,1500000,1000000,500000
2023-01-08,18000000,2500000,1800000,1200000,600000
...
```

**요구 사항:**

  - 최소 52주 데이터 (104주 권장)
  - 주간 집계 (월요일-일요일)
  - 주(Week)당 하나의 행(Row)
  - 매출과 지출은 동일한 통화 단위 사용

### 2단계: 데이터 업로드

1.  **📊 Data Upload(데이터 업로드)** 페이지로 이동
2.  "Browse files"를 클릭하여 CSV 선택
3.  컬럼 매핑:
      - 날짜 컬럼: `date`
      - 매출 컬럼: `revenue`
      - 미디어 채널: 모든 지출(spend) 컬럼 선택
4.  데이터 품질 보고서 검토
5.  "Save Configuration(구성 저장)" 클릭

### 3단계: 모델 구성

1.  **⚙️ Model Config(모델 구성)** 페이지로 이동
2.  각 채널의 변환 파라미터 설정:

**Adstock (이월 효과):**

  - TV/브랜드: 0.6-0.8 (오래 지속되는 효과)
  - 디지털 퍼포먼스: 0.2-0.4 (단기 효과)
  - 소셜 미디어: 0.3-0.6 (중기 효과)

**Hill Saturation (포화도):**

  - K (Scale/규모): 보통 1.0 (여기서 시작)
  - S (Shape/형태): 보통 1.0 (여기서 시작)

<!-- end list -->

3.  "Train Model(모델 학습)" 클릭

### 4단계: 결과 분석

1.  **📈 Results(결과)** 페이지로 이동
2.  다음 항목 검토:
      - 채널별 기여도 (폭포수 차트)
      - 채널별 ROAS (광고 수익률)
      - 반응 곡선 (포화 지점 확인)
      - 모델 진단 지표 (R², MAPE)

### 5단계: 예산 최적화

1.  **💰 Budget Optimizer(예산 최적화)** 페이지로 이동
2.  총 예산 입력
3.  채널별 제약 조건 설정 (선택 사항)
4.  "Optimize Budget(예산 최적화)" 클릭
5.  최적화된 배분 vs 현재 배분 비교

## Data Preparation (데이터 준비)

### 데이터 형식 요구 사항

**최소 요구 사항:**

  - **52주** 데이터 (1년)
  - **주간 집계** (일별 데이터는 노이즈가 너무 많음)

**최적 조건:**

  - **104-156주** (2-3년)
  - **누락된 주 없음** (공백은 0으로 채우거나 보간법 사용)
  - **외생 변수 포함** (계절성, 프로모션, 휴일 등)

### 컬럼 명명법

**필수 컬럼:**

```
date          - 형식: YYYY-MM-DD
revenue       - 총 매출 (또는 설치 수, 전환 수)
[channel1]    - 채널 1 지출 (예: google_uac)
[channel2]    - 채널 2 지출 (예: meta)
...
```

**선택 컬럼:**

```
promotion     - 프로모션 주간 표시 (이진값 0/1)
seasonality   - 계절성 (분기별 1-4 숫자 등)
holiday       - 휴일 주간 표시 (이진값)
installs      - 모바일 게임의 경우 (보조 KPI)
```

### 다중 시장(Multi-Market) 데이터 형식

글로벌 캠페인의 경우 다음 형식을 사용하세요:

```csv
date,country,os,revenue,installs,google_uac,meta
2023-01-01,US,iOS,5000000,1000,800000,600000
2023-01-01,US,Android,4000000,1500,700000,500000
2023-01-01,KR,iOS,3000000,800,500000,400000
...
```

**추가 컬럼:**

  - `country` - 2자리 국가 코드 (US, KR, JP 등)
  - `os` - 플랫폼 (iOS, Android)

### 데이터 품질 체크

업로드 전 확인 사항:

✅ **중복 날짜 없음** (각 주는 한 번만 등장)
✅ **극단적인 이상치 없음** (블랙 프라이데이 같은 실제 상황 제외)
✅ **일관된 스케일링** (백만 단위와 천 단위를 섞지 말 것)
✅ **음수 값 없음** (지출과 매출은 0 이상이어야 함)

## Model Configuration (모델 구성)

### Adstock (이월 효과) 이해하기

**Adstock이란?**
광고의 지연되고 지속되는 효과입니다. 이번 주에 TV 광고에 10억 원을 썼다면, 그 영향은 몇 주간 지속됩니다.

**Decay Rate(감쇠율) 파라미터 (0-1):**

  - **0.0**: 이월 효과 없음 (이번 주에만 효과 발생)
  - **0.5**: 이번 주 효과의 50%가 다음 주에도 지속됨
  - **0.8**: 80%가 이월됨 (매우 오래 지속)

**선택 가이드:**

| 채널 유형 | 추천 감쇠율 | 이유 |
|-----------|-------------|------|
| TV/브랜드 | 0.6-0.8 | 장기적인 브랜드 구축 효과 |
| 검색 (Google) | 0.1-0.3 | 즉각적인 퍼포먼스 |
| 소셜 (Meta) | 0.3-0.6 | 중기적인 인게이지먼트 |
| 디스플레이 | 0.4-0.7 | 리타겟팅 효과 |
| 인플루언서 | 0.5-0.8 | 콘텐츠가 계속 노출됨 |

**예시:**

```
1주 차: 지출 $100 → 효과 $100
2주 차: 지출 $0   → 효과 $50 (1주 차의 50% 감쇠)
3주 차: 지출 $0   → 효과 $25 (2주 차의 50% 감쇠)
```

### Hill Saturation (포화도) 이해하기

**Saturation(포화)이란?**
지출을 늘릴수록 수익 효율이 떨어지는 현상(수확 체감)입니다. 처음 10억 원의 효과가 열 번째 10억 원의 효과보다 큽니다.

**파라미터:**

  - **K (Scale/규모)**: 달성 가능한 최대 효과

      - 보통 1.0-2.0
      - K가 높을수록 잠재적 임팩트가 큼

  - **S (Shape/형태)**: 포화가 얼마나 빨리 일어나는가

      - S = 1.0: 부드러운 곡선 (기본값)
      - S \< 1.0: 완만한 포화 (오랫동안 효율 유지)
      - S \> 1.0: 가파른 포화 (빠른 효율 감소)

**선택 가이드:**

모든 채널에 대해 **K=1.0, S=1.0**으로 시작한 후 다음을 기준으로 조정하세요:

  - 과거 실험을 통해 알려진 포화 지점
  - 산업 벤치마크
  - 비즈니스 직관

### 모델 설정

**Ridge Alpha (정규화 강도):**

  - **낮음 (0.1-0.5)**: 정규화가 적음, 과적합(Overfitting) 가능성 있음
  - **중간 (1.0)**: 균형 잡힘 (권장 기본값)
  - **높음 (5.0-10.0)**: 강한 정규화, 모델이 더 안정적임

**Train/Test Split (학습/테스트 분할):**

  - **80/20**: 표준 (80% 학습, 20% 테스트)
  - 테스트 셋을 사용하여 모델 정확도 검증

## Understanding Results (결과 해석)

### Channel Contributions (채널 기여도) 탭

**Waterfall Chart (폭포수 차트):**
각 채널이 전체 매출에 어떻게 기여하는지 보여줍니다.

```
Base (자연 유입): ████████ 20%
Google UAC:       ██████████████ 30%
Meta:             ████████████ 25%
Apple Search:     ████████ 15%
Others:           ██████ 10%
───────────────────────────────
Total Revenue:    ████████████████████ 100%
```

**주요 지표:**

| 지표 | 의미 | 좋은 값 |
|------|------|---------|
| **ROAS** | 지출 1원당 매출 | 퍼포먼스 채널의 경우 \> 2.0 |
| **Contribution %** | 전체 매출 중 비중 | 전략과 일치해야 함 |
| **Spend** | 총 투자 금액 | 해당 없음 |

### Response Curves (반응 곡선) 탭

**확인해야 할 사항:**

✅ **건강한 곡선:**

```
매출 ▲
    │     ╭──── (포화됨)
    │   ╭─╯
    │ ╭─╯
    │╭╯
    └──────────► 지출
```

❌ **문제: 이미 포화됨 (Already Saturated)**

```
매출 ▲
    │ ────────── (평평함)
    │╭──────────
    │╯
    └──────────► 지출
      ↑
   현재 지출
```

**조치:** 해당 채널의 예산을 줄이세요.

❌ **문제: 포화가 보이지 않음 (No Saturation Visible)**

```
매출 ▲
    │          ╱
    │        ╱
    │      ╱
    │    ╱
    └──────────► 지출
```

**조치:** 데이터 불충분 또는 모델 적합도가 낮을 수 있습니다.

### Model Diagnostics (모델 진단) 탭

**R² (결정 계수): 0.0-1.0**

  - **\> 0.7**: 훌륭한 적합도
  - **0.5-0.7**: 좋은 적합도
  - **\< 0.5**: 낮은 적합도 (데이터 품질 확인 필요)

**MAPE (평균 절대 비율 오차):**

  - **\< 10%**: 매우 정확함
  - **10-20%**: 양호함
  - **\> 20%**: 정확도 낮음

**모델 적합도가 낮다면?**

1.  데이터 품질 확인 (이상치, 결측치)
2.  외생 변수 추가 (계절성, 프로모션)
3.  Adstock/Hill 파라미터 변경 시도
4.  더 많은 데이터 수집 (이상적으로 2년 이상 필요)

## Budget Optimization (예산 최적화)

### 최적화 도구 사용하기

**1단계: 총 예산 설정**

```
희망 총 예산 입력: $5,000,000
```

**2단계: 제약 조건 설정 (선택 사항)**

| 채널 | 최소 지출 | 최대 지출 | 이유 |
|------|-----------|-----------|------|
| Google | $1M | $3M | 계약상 약정 |
| Meta | $500K | $2M | 테스트 예산 상한 |
| Apple | $0 | $2M | 유동적 |

**3단계: 최적화 실행**

"Optimize Budget"을 클릭하여 확인:

**현재 배분 (Current Allocation):**

```
Google:  $2.0M  (ROAS 3.0)
Meta:    $1.5M  (ROAS 2.5)
Apple:   $1.5M  (ROAS 2.2)
────────────────────────────
Total:   $5.0M  (Revenue: $14M)
```

**최적화된 배분 (Optimized Allocation):**

```
Google:  $2.5M  (ROAS 2.8) ← +$500K
Meta:    $1.8M  (ROAS 2.6) ← +$300K
Apple:   $0.7M  (ROAS 3.0) ← -$800K
────────────────────────────
Total:   $5.0M  (Revenue: $15.2M)
                      ↑
                   +8.6% 개선!
```

### 시나리오 빌더 (Scenario Builder)

**사용 사례: "예산을 20% 삭감한다면?"**

1.  슬라이더 조정:

      - Google: $2.0M → $1.6M
      - Meta: $1.5M → $1.2M
      - Apple: $1.5M → $1.2M

2.  예측 영향 확인:

      - Total: $5.0M → $4.0M
      - Revenue: $14.0M → $12.5M
      - ROAS: 2.8 → 3.1 (효율성 개선\!)

3.  "보수적 Q1"으로 저장

4.  여러 시나리오 비교:

      - 현재 (Current)
      - 보수적 Q1 (Conservative Q1)
      - 공격적 성장 (Aggressive Growth)
      - 구글 집중 (Google Focus)

### 추천 사항 해석하기

**최적화 도구가 큰 변화(\>30%)를 제안할 경우:**

  - ⚠️ **한 번에 모두 적용하지 마세요**
  - ✅ 추천 방향으로 10-20%만 이동
  - ✅ 2-4주간 테스트
  - ✅ 새로운 데이터로 모델 다시 실행
  - ✅ 점진적으로 반복

**왜 점진적으로 적용해야 하나요?**

  - 모델은 과거 패턴을 기반으로 합니다
  - 시장 상황은 변합니다
  - 진행 중인 캠페인의 급격한 중단을 방지합니다
  - 학습하며 조정해야 합니다

## Multi-Market Analysis (다중 시장 분석)

### 다중 시장 모델 설정

**사용 시기:**

  - 2개 이상의 국가에서 캠페인 운영 시
  - iOS와 Android의 경제성이 다를 때
  - 국가별 인사이트가 필요할 때

**분석 레벨:**

| 레벨 | 사용 사례 | 예시 |
|------|-----------|------|
| **Global** | 단일 통합 모델 | 예산이 적고 유사한 시장들 |
| **By Country** | 국가별 인사이트 | US, KR, JP의 ROAS가 다름 |
| **By OS** | 플랫폼 차이 | iOS가 Android보다 LTV가 높음 |
| **Country × OS** | 세부 제어 | US-iOS와 US-Android가 다름 |

### 시장 비교

**히트맵(Heatmap) 뷰:**

```
       Google  Meta  Apple
US     3.5     3.2   2.8
KR     2.8     2.5   2.0
JP     3.2     2.8   2.5
```

**인사이트:**

  - Google은 US에서 가장 성과가 좋음
  - KR에서는 모든 채널이 약세 (경쟁이 치열한 시장?)
  - JP는 일관된 성과를 보임

### 시장 간 인사이트 (Cross-Market Insights)

**질문: "새로운 국가로 확장해야 할까요?"**

유사한 국가를 비교하세요:

  - 만약 KR ROAS = 2.5이고 TW(대만)가 유사한 시장이라면 → TW ROAS \~2.3-2.7 예상
  - 테스트 단계 예산을 이에 맞춰 책정

## Best Practices (모범 사례)

### 데이터 수집

✅ **할 것 (Do):**

  - 모든 마케팅 지출을 일관되게 추적하세요
  - 기여(Attribution) 추적을 위해 UTM 파라미터를 사용하세요
  - 매주 같은 요일에 데이터를 추출하세요
  - 캠페인/이벤트를 문서화하세요 (데이터와 병합)
  - 2-3년 치 기록을 유지하세요

❌ **하지 말 것 (Don't):**

  - 서로 다른 기여 기간(window)을 섞지 마세요
  - 추적 방법론을 중간에 바꾸지 마세요
  - 자연(Organic/Base) 매출을 무시하지 마세요
  - "나쁜" 주간 데이터를 삭제하지 마세요 (모든 데이터 유지)

### 모델 개발

✅ **할 것 (Do):**

  - 단순한 모델로 시작하세요 (기본 파라미터 사용)
  - 홀드아웃(Holdout) 테스트 셋으로 검증하세요
  - 예측값과 실제 결과를 비교하세요
  - 가정과 결정을 문서화하세요
  - 데이터가 쌓이면 분기별로 재학습하세요

❌ **하지 말 것 (Don't):**

  - 완벽하게 맞추기 위해 파라미터를 과도하게 조정하지 마세요 (과적합)
  - 지출 기간이 10주 미만인 채널에 모델을 사용하지 마세요
  - 모델 진단 지표(R², MAPE)를 무시하지 마세요
  - 비즈니스 감각 없이 모델을 맹신하지 마세요

### 결과 활용

✅ **할 것 (Do):**

  - 추천 사항을 점진적으로 테스트하세요 (10-20% 이동)
  - MMM과 순증분(Incrementality) 테스트를 병행하세요
  - 이해관계자와 결과를 공유하세요 (시각화)
  - 실제 성과 vs 예측을 모니터링하세요
  - 반복하고 개선하세요

❌ **하지 말 것 (Don't):**

  - 모델만 믿고 예산을 50% 이상 급격히 변경하지 마세요
  - 실험을 중단하지 마세요 (A/B 테스트는 MMM을 보완합니다)
  - 완벽한 예측을 기대하지 마세요 (모델은 가이드입니다)
  - 시장 변화에 따라 모델 업데이트하는 것을 잊지 마세요

## Troubleshooting (문제 해결)

### "모델 적합도가 낮습니다 (R² \< 0.5)"

**가능한 원인:**

1.  데이터 부족 (52주 이상 필요)
2.  데이터 대비 너무 많은 채널 수
3.  기간 중 주요한 시장 변화 발생
4.  핵심 변수 누락 (계절성, 이벤트)

**해결책:**

1.  데이터 추가 수집
2.  유사한 채널 통합
3.  기간 분할 (변화 전/후)
4.  외생 변수 추가

### "최적화가 극단적인 배분을 제안합니다"

**예시: "Google에 90%, 나머지에 10% 할당"**

**원인:**

  - 모델이 보기에 Google의 과거 ROAS가 가장 높았음
  - 과거 지출 범위를 벗어난 포화도를 고려하지 않음
  - 선형적 확장(Linear scaling)을 가정함 (잘못된 가정)

**해결책:**

1.  보수적인 제약 조건 설정 (채널별 최소/최대)
2.  점진적 실행 (먼저 20% 이동 테스트)
3.  Hill 파라미터가 너무 약한지 확인 (S값 증가)
4.  비즈니스 판단으로 재조정

### "결과가 제 직관과 맞지 않습니다"

**예시: "TV ROAS가 낮게 나왔지만, 브랜드 구축에 중요하다고 생각함"**

**가능한 이유:**

1.  브랜드 효과가 다르게 측정됨 (장기 LTV가 데이터에 없음)
2.  후광 효과(Halo effect)가 포착되지 않음 (TV가 검색을 부스팅함)
3.  데이터 품질 문제 (추적 오류)
4.  Adstock이 너무 낮음 (TV 효과는 장기적임)

**해결책:**

1.  TV Adstock 증가 (0.7-0.8)
2.  브랜드 지표 추가 (인지도, 고려도 등)
3.  TV 추적 정확도 검증
4.  TV에 대한 순증분 테스트 고려

### "라스트 클릭(Last-click) 기여와 다릅니다"

**이것은 정상입니다\!** MMM과 라스트 클릭은 서로 다른 것을 측정합니다:

| 지표 | 라스트 클릭 (Last-Click) | MMM |
|------|--------------------------|-----|
| **무엇을** | 어떤 터치포인트가 전환을 유도했나 | 전체적인 순증분(Incremental) 영향 |
| **관점** | 바텀 퍼널 (Bottom-funnel) | 풀 퍼널 (Full-funnel) |
| **편향** | 검색, 브랜드에 유리 | 더 균형 잡힘 |

**둘 다 유용합니다:**

  - 라스트 클릭: 전술적 최적화
  - MMM: 전략적 예산 배분

## Glossary (용어 사전)

**Adstock (애드스탁)**: 시간 경과에 따른 광고의 이월(잔존) 효과

**ROAS (광고 수익률)**: 광고비 1달러(또는 1원)당 발생한 매출

**Saturation (포화)**: 추가 지출이 수확 체감(효율 감소)을 일으키는 지점

**Ridge Regression (릿지 회귀)**: 과적합을 방지하기 위해 정규화를 사용하는 통계적 방법

**Incrementality (순증분)**: 마케팅으로 인해 발생한 추가 매출 (마케팅이 없었더라도 발생했을 매출 제외)

**Base/Organic (베이스/자연 유입)**: 유료 마케팅 없이 발생하는 매출

**Exogenous Variables (외생 변수)**: 매출에 영향을 미치는 외부 요인 (계절성, 휴일, 이벤트)

**R² (결정 계수)**: 모델이 매출의 변동성을 설명하는 비율 (0-1 척도)

**MAPE**: 평균 절대 비율 오차 (예측 오차의 평균 %)
